TS类型

1. string(字符串)​

  • 与JavaScript一样，可以使用双引号（"）或单引号（'）表示字符串。
  const str:string = 'Hello TypeScript'

  // 将数值类型分配给str，会报错
  str = 18  // Error 不能将类型“number”分配给类型“string”。
  • 也可以使用ES6的模板字符串
  let str: string = 'learning TypeScript'
  str = `Welcome to ${str}`
  // output> Welcome to learning TypeScript

2. number(数字)​

  • 支持十进制和十六进制字面量,也支持二进制和八进制字面量
  let decLiteral: number = 0 // 十进制 output> 0
  let hexLiteral: number = 0xff // 十六进制 output> 255
  let binaryLiteral: number = 0b11 // 二进制 output> 3
  let octalLiteral: number = 0o11 // 八进制 output> 9

3. boolean(布尔值)​

  • 就是简单的true/false值，其他语言中布尔值都是一样。
  let bool: boolean = true

4. Array(数组)​

  • TypeScript也可以操作数组元素，有两种定义方式
    • (1) 元素类型[ ]的方式
      • 表示由该类型组成的数组，数组元素不能有其他类型
      const arr: number[] = [1, 2, 3]
      // 如下，定义了一个number数组，数组元素出现了一个string类型
      const arr1: number[] = [1, 2, '3'] // output> 保错，不能将类型“string”分配给类型“number”。
    • (2) 使用泛型 Array<元素类型>
      • 表示由该类型组成的数组，数组元素不能有其他类型
      let list: Array<string> = ['Tom', 'Jack']

5. Tuple(元组)​

  • 用来存储已知元素数量和类型的数组，元素和类型一一对应。
  const tuple: [number, boolean] = [1, true] 
  console.log(tuple[0]) // output> 1
  
  // 元素位置要跟类型声明的位置对应，否则报错
  const tuple: [number, boolean] = [true, 1] // Error
  • 元组可以使用es6的解构赋值
  const tuple: [number,string] = [2, '2']
  const [a, b] = tuple // output> a = 2, b = '2'

6. enum(枚举)​

  • 是对JavaScript类型的一个补充，用于定义某一个集合
  • 默认情况下，元素的下标从0开始
  enum Direction {
    Top,
    Right,
    Bottom,
    Left,
  }
  let direction: Direction = Direction.Left  // output> 3
  • 可以手动修改某一个元素或者全部的下标
  enum Direction {
    Top = 1,
    Right,
    Bottom = 4,
    Left,
  }
  let direction: Direction = Direction.Right // output> 2
  let direction: Direction = Direction.Left // output> 5
  • 同样可以通过索引来查看它对应的名字
  enum Direction {
    Top = 1,
    Right,
    Bottom = 4,
    Left,
  }
  let direction: string = Direction[1] // output> Top

7. Any(任意值)​

  • 当我们对类型不明确的变量使用的一种数据类型，如下三种情况可以使用any:
    • 变量的值会动态改变时，比如来自用户的输入：
        let x: any = 1;    // number
        x = 'I am who I am';    // string
        x = false;    // boolean
    • 存储各种类型数据的数组：
        const arrayList: any[] = [1, false, 'fine'];

8. unknown​

  • TypeScript 3.0中引入的 unknown 类型也被认为是 top type ,被认为更安全
  • 与 any 一样，所有类型都可以分配给unknown，比any更加严格
  //unknown 可以定义任何类型的值
  let unk: unknown
 
  unk = true;             // output> OK
  unk = 42;               // output> OK
  unk = "Hello World";    // output> OK
  unk = [];               // output> OK
  unk = {};               // output> OK
  unk = null;             // output> OK
  unk = undefined;        // output> OK
  unk = Symbol("type");   // output> OK
  • 下面写会报错，因为unknown类型不能作为子类型只能作为父类型,但是any可以作为父类型和子类型
  // unknown类型不能赋值给其他类型
  let str: unknown = '123'
  let str1: string = str // output> 报错
 
  // 下面就没问题 any类型是可以的
  let str: any = '123'
  let str1: string = str   
 
  // unknown可赋值对象只有unknown 和 any
  let str: unknown = '123'
  let str1: any = '456'
  str = str1
  • 另外，any定义的对象，访问对象中没有的属性，不会报错
  • unknown定义的对象，不能调用属性和方法
  // any类型在对象没有这个属性的时候还在获取是不会报错的
  let obj: any = { b: 1 }
  console.log(obj.a)
 
  // 如果是unknown 是不能调用属性和方法
  let obj: unknown = { b: 1 }
  console.log(obj.b) // output> “obj”的类型为“未知”

9. Void​

  • 当一个函数没有返回值的时候，返回类型为void
  function fn(): void {
    console.log(111)
  }

10. Null和Undefined​

  • 在js中,null是一个只有一个值的特殊类型。表示一个空对象引用，用 typeof 检测 null 返回是 object。
  • 在js中, undefined 是一个没有设置值的变量，typeof 一个没有值的变量会返回 undefined。
  • Null 和 Undefined 是其他任何类型（包括 void）的子类型，可以赋值给其它类型
  • 在TypeScript中启用严格的空校验（--strictNullChecks）特性，就可以使得null 和 undefined 只能被赋值给 void 或本身对应的类型
  // 启用 --strictNullChecks
  let x: number;
  x = 1; // 编译正确
  x = undefined;    // 编译错误
  x = null;    // 编译错误
  • 如果一个类型可能出现 null 或 undefined， 可以用 | (联合类型)来支持多种类型
  // 启用 --strictNullChecks
  let x: number | null | undefined;
  x = 1; // 编译正确
  x = undefined;    // 编译正确
  x = null;    // 编译正确
  • void、null和undefined的区别
    • 与 void 的区别是，undefined 和 null 是所有类型的子类型，可以赋值给其他类型
  • 如果开启了严格模式，null不能赋给void类型
  {
    "compilerOptions":{
        "strict": true
    }
  }

  const n: void = null // output> 会报错，不能将类型“null”分配给类型“void”

11. Never​

  • never 是其它类型（包括 null 和 undefined）的子类型，代表从不会出现的值
  • 返回never类型的函数有：
  // 返回never的函数必须存在无法达到的终点
  function error(message: string): never {
    throw new Error(message);
  }

  // 推断的返回值类型为never
  function fail() {
    return error("Something failed");
  }

  // 返回never的函数必须存在无法达到的终点
  function infiniteLoop(): never {
    while (true) {}
  }

12. 类型断言​

  • 一些情况下，我们比TypeScript更了解某个值，我们可以使用类型断言，来断定值的类型
  • 类型断言有两种方式：
  • 两种形式是等价的。 至于使用哪个大多数情况下是凭个人喜好；然而，当你在TypeScript里使用JSX时，只有as语法断言是被允许的。
    • “尖括号”语法：
      const type: any = 'string'
      const length: number = (<string>type).length
    • as 语法
      const type: any = 'string'
      const length: number = (type as string).length

13. 联合类型​

  • 有时候，我们某个变量可能是数值，也可能是字符串，那么可以使用联合类型
  let abc: number | string
  abc = 123   // output> OK  
  abc = '123'  // output> OK
